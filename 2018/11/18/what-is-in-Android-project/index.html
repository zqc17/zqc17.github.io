从零开始--Android工程基本结构

发表于 2018-11-18 | 分类于 Android

本文简单介绍一个Android工程的基本构成，参考菜鸟教程。

Android工程结构

下图是Android开发过程中主要关注的部分。其中src/main/java存放Java写的业务代码，src/main/res存放字符串及各种多媒体资源。

res—资源文件夹

主要包含以下资源：

  • 图片。drawable, mipmap-hdpi, mimap-mdpi, mipmap-xhdpi, mimap-xxhdpi 这些文件夹下面存放着图片资源。
  • 布局。layout 文件夹下面存放着布局资源。
  • values。 values 文件夹下面存放着 demens.xml, string.xml, colors.xml, arrays.xml, attrs.xml 等资源。

Android工程中的三个主要文件

MainActivity.java, activity_main.xml, AndroidManifest.xml 分别是主界面对应的程序，布局和整个APP的配置文件。

MainActivity.java包含MainActivity类，Android app中主界面的加载和主要的用户逻辑都在这里实现。

activity_main.xml包含了MainActivity类需要引用到的布局资源，也即主界面的UI设计。

AndroidManifest.xml需要Android app中包含的所有Activity以及需要的权限，如读取联系人、获取位置信息等权限。
