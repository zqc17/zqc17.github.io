Android NDK 入门 - 环境配置

发表于 2020-06-25 | 分类于 Android

本文介绍 Android NDK 入门的环境配置。

Android NDK 安装

Android Studio 打开 File > Settings > Appearance&Behavior > System Settings > Android SDK，在 SDK Tools 标签页中下载 NDK 和 CMake。

下载完成之后需要配置 NDK 路径，有以下几种配置方式可选：

  1. 通过 File > Other Settings > Default Project Structure 配置 NDK 路径， Windows 下一般路径位于 C:\Users\<用户名>\AppData\Local\Android\Sdk\ndk\<NDK版本>。
  2. 通过 File > Project Strucutre 配置 NDK 路径，这个配置只针对当前打开的工程。
  3. 编辑当前工程目录下的 local.properties 配置 NDK 路径(ndk.dir=<路径>)，效果和 2. 相同。

CMake文件简介

新建的 NDK 工程默认采用 CMake 作为 C++ 的构建脚本，这里简单对 CMake 进行介绍。

CMake 是跨平台的make工具（Cross-platform make），CMake构建脚本可以针对具体的构建目标，翻译成对应的make工具可以识别的构建脚本，并调研对应的make工具完成构建。

常见的make例如Linux下的GNU make，Qt的qmake，微软的nmake。
